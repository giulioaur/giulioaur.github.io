Giulio Auriemma

Computer Scientist & Game Developer

I'm Giulio, a young computer scientist who loves programming. As all real programmers do, I started learning C (the Lord of programming language), and during my university education I have learned to use it both in high level applications and in Linux low level ones. Then I learned about the object-oriented paradigm, starting from C++ until Java. Since I was child, I have always seen the web as a magic world, in which all is possible, so when I was able to understand it, I started studying "web languages". Both for academic reasons and for personal knowledge I have always tried different kind of languages to write applications, so I know the most famous language and many framework, but pay attention, as I told you, I'm a computer scientist, not a programmer. The main difference is that the first one can analyze your problem, find the optimal solution, choose between many tools for the best one, design a software through smart choices, write code, debug it and produce good documentation to correctly maintain it; the second one just writes the code you want, probably in a faster way, but not in a better one. This is like you have written a programming language and you must choose between an approach with compiler, optimizer and interpreter and the one with the interpreter only. I see programming more as a game than as a work: every project is a level in which you must defeat some mini-boss (the bugs) using your weapon (your knowledge) and your magic book (StackOverflow) until you save the princess (your program) and gain her kiss ( I would say it is something like personal satisfaction or gratification, but we all know that it represent money). Sometimes the level is timed (the tremendous timelines), but if you hate the laggard as much as I do, it will be not a problem. But remember: if you can beat a monster with a jump, you don't need to do a triple somersault, as Ockham said: "Between many solutions, the simplest is the best one".
